# Manual Testing Project – Guru99 Demo Site

## 👤 Project Overview
This project demonstrates manual software testing of the Guru99 Banking Demo Website.  
The goal is to practice real-world QA activities including test case design, execution, and defect reporting.

---

## 🎯 Objectives
- Understand requirements and system behavior
- Design effective test cases and test scenarios
- Perform functional and non-functional testing
- Execute manual test cases
- Report and document defects clearly

---

## 🌐 Application Under Test
- Website: Guru99 Demo Banking Application  
- Type: Web-based banking simulation system

---

## 🧪 Testing Activities Performed

### 1. Test Case Design
- Created test cases based on functional requirements
- Covered positive, negative, and boundary scenarios

### 2. Test Scenario Design
- Identified main modules such as:
  - Login functionality
  - New customer creation
  - Account operations

### 3. Test Execution
- Executed manual test cases step by step
- Compared actual results with expected results

### 4. Defect Reporting
- Documented bugs with:
  - Steps to reproduce
  - Expected result
  - Actual result
  - Severity and priority

---

## 🧰 Testing Types Used
- Functional Testing
- Negative Testing
- Boundary Value Testing
- Black-box Testing Techniques

---

## 📚 Skills Gained
- Understanding of STLC (Software Testing Life Cycle)
- Writing structured test cases
- Identifying defects in web applications
- Improving attention to detail in QA process

- Browser testing tools
